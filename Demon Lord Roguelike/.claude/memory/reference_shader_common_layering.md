Shader 内 `#include` 的公共 `.hlsl` 按"是否跨效果通用"分层存放：

- **`Assets/FrameWork/Shader/Common/`** = 跨效果通用基础设施库。例：`ParticleCommon.hlsl`（URP 透明粒子通用件：`TEXTURE2D(_BaseMap)` 采样器 + `PARTICLE_COMMON_CBUFFER` 公共材质字段宏 + `ParticleFade(screenPos, softNear, softFar, camNear, camFar)` 柔和粒子/相机淡出函数）；`Noise.hlsl`（程序化噪声, 2026-07 新增：`Hash21(p)` 2D→1D 哈希 + `ValueNoise2D(p)` 值噪声(平滑曲线双线性插值, 输入每+1换一格故用 scale 控颗粒大小) + `Fbm2D(p, octaves)` 分形叠加(每层频率×2/振幅÷2, 除以总振幅归一化到[0,1], octaves 须编译期常量因循环被 unroll)。纯函数、不引用全局 uniform、不依赖噪声贴图。选值噪声而非 Perlin/Simplex=火焰/烟雾这类湍流只需连续无重复感, 指令数明显更少。首个用户=火球核心的火焰湍流）。
- **`Assets/FrameWork/Shader/Effect/<效果名>/`** = 效果专属业务件。例：`Effect/WindSway/GrassWind.hlsl`、`TreeWind.hlsl`（草/树风摆的 CBUFFER 风摆字段 + `ApplyWind` 算法：草=`pow(uv.y,_Stiffness)`硬度+`_WindDir`顺风弯倒；树=`_AnchorBottom`锚点+树冠下压）。

**判定标准（术语校准结论）**：「被多个 shader 复用 ≠ 通用」。只有"跟具体效果解耦、能被无关效果直接 include 复用"的才进 Common。检验法：「一个无关粒子效果(如火花)能拿来即用吗？」能→Common，不能→效果目录。`Effect/` 不是"粒子专用"目录，`Effect/<效果>/` 是该效果专属目录，业务件就近放=高内聚，Common 保持纯净。

**风摆粒子 2 shader 落地结构(2026-07 合并)**：原 `Shader_Particle_{Grass,Tree}WindSway_{Unlit,Lit}.shader` 4 个**已合并为 2 个** `Shader_Particle_{Grass,Tree}WindSway.shader`(去掉 _Lit/_Unlit 后缀，内部名 `FrameWork/Particle/{Grass,Tree}WindSway`)，用 `[ToggleOff(_UNLIT_ON)]_LitEnable` 在单 shader 内切换受光/无光(详见下方"WindSway Lit 合并段")。include 链 `.shader → Effect/WindSway/{Grass,Tree}Wind.hlsl → ../../Common/ParticleCommon.hlsl`。改通用淡出=改 `ParticleCommon`(1处全生效)；改草/树风摆=改对应 `*Wind.hlsl`。注：草粒子 shader 除 UniversalForward 外带 **ShadowCaster + DepthOnly** 两 pass(复用 `ApplyWind`+贴图 alpha `clip`，不加 URP 的 ApplyShadowBias)；树粒子仅单 forward pass。草粒子受光/阴影整套踩坑见 [[reference_grass_particle_lit_shadow]]。

**约束**：①`PARTICLE_COMMON_CBUFFER` 宏须在引用方 `CBUFFER_START(UnityPerMaterial)` 块首展开、其后接 shader 特有字段，保证所有 material uniform 在单一 UnityPerMaterial 块内(SRP Batcher 兼容)。②ShaderLab 的 Properties 块无法 `#include`，每个 `.shader` 仍各留一份参数声明（改动频率低）。③`.hlsl` include 用相对路径(`../../Common/...`)，`.hlsl.meta` 用 `ShaderIncludeImporter`。

**Alpha 扩张描边(2026-07 新增, 6 个 WindSway shader 全覆盖)**：`Common/Outline.hlsl`=跨效果通用描边件(`ApplyAlphaOutline(TEXTURE2D_PARAM(tex,samp), baseCol, uv, texel, size, color)`：采样8邻域最大alpha，在精灵透明区沿轮廓外扩描边色；纹理用 `TEXTURE2D_PARAM` 参数化传入，不引用具体贴图名，故任意用 `_BaseMap` 的透明/裁剪 shader 均可复用)。选 alpha 扩张而非法线外扩：草/树是固定法线 billboard/精灵，法线外扩会错乱。每个 shader 接线三件套：Properties 加 `[Header(Outline)] [Toggle(_OUTLINE_ON)]_OutlineEnable(默认0)/[HDR]_OutlineColor/_OutlineSize(纹素数 Range0-10)`；正向 pass 加 `#pragma shader_feature_local _OUTLINE_ON`；frag 采样后 clip 前 `#if defined(_OUTLINE_ON) baseSample=ApplyAlphaOutline(...)`。CBUFFER 三字段 `float4 _BaseMap_TexelSize(Unity按贴图自动填,放UnityPerMaterial内保SRP Batcher)/half4 _OutlineColor/half _OutlineSize`——粒子2个走 `PARTICLE_COMMON_CBUFFER` 宏(改1处2生效)+ParticleCommon 里 include Outline；Mesh/Sprite 各自 inline CBUFFER 手加 + `#include "../Common/Outline.hlsl"`。**要点**：描边须在 `clip` 前叠加(否则描边像素被裁掉)；Lit 变体描边随采样进 albedo 会被光照(可接受)；shadow/depth pass 不外扩(描边不投影/不写深度)；精灵贴图须留透明 padding 才扩得出描边。

**统一材质面板 `WindSwayShaderGUI`**(`Effect/WindSway/Editor/WindSwayShaderGUI.cs`, 无 asmdef→进 Assembly-CSharp-Editor, 全局类名 `CustomEditor "WindSwayShaderGUI"` 解析)：轻量分组折叠 GUI(非 PaletteFX 图片页签重型版)。按 表面/光照/描边/柔和粒子/相机淡出/位置偏移/风摆/抖动/阴影 分组的 `Section[]`(全 shader 参数并集), "属性存在才画"自适应 6 个不同参数集；一级开关(Lit/Outline/Soft/Camera 的 `_*Enable`)用 `ShaderProperty` 自动切 keyword、关闭时组内置灰；"光照"组是只有 `_LitEnable` 开关无子参数的空组(present.Count=0 但 toggleProp≠null 仍绘制)；**表面类型/渲染模式/Alpha 裁剪(_Cutoff)/渲染面 已从本 GUI 的 Section 移除, 改由 `SurfaceOptionsGUI.Draw/Sync` 统一绘制成"渲染设置"折叠组并同步混合/深度/队列**(见下方 SurfaceOptions 段的 WindSway 接入)——原"粒子混合(_BlendSrc/_BlendDst/_ZWrite)"与"渲染面(_Cull)"两 Section 已删；折叠态存 EditorPrefs；末尾 RenderQueueField+EnableInstancingField；兜底"其他"组防漏显。6 个 shader 均已挂 `CustomEditor "WindSwayShaderGUI"`。

**通用受光件 `Common/CommonLit.hlsl`(2026-07 新增)**：跨效果通用 Lit 助手，`ApplyCommonLit(albedo, alpha, positionWS, normalWS, positionHCS, fogFactor)` 内部封装 `SurfaceData`+`InputData`+`UniversalFragmentBlinnPhong`+`MixFog`，返回受主光/附加光/阴影/环境光(SH)照亮并混雾的最终色；文件头自带 `#include .../Lighting.hlsl`(带 SurfaceData/BlinnPhong)，故引用方无需另 include 光照库。任意"是否开启 Lit"的 shader 均可复用。

**WindSway Lit 合并段(2026-07)——`[ToggleOff(_UNLIT_ON)]` 零材质迁移**：给"本已受光/已有材质引用"的 shader 加 Lit 开关时，**绝不能**用普通 `[Toggle(_LIT_ON)]`(无关键字=无光)——老材质 `m_ValidKeywords: []` 空关键字会瞬间变无光。正解：用 **`[ToggleOff(_UNLIT_ON)] _LitEnable`**(勾选=float1=关键字**关**=受光；取消=float0=关键字**开**=无光)，使"无关键字=受光"，所有老材质(空关键字)自动保持受光、**0 个材质需要改动**。落地：①粒子草/树把 Lit+Unlit **合并为单 shader**——正向 pass 加 `#pragma shader_feature_local _UNLIT_ON` + 全套阴影/附加光 multi_compile；Varyings 取受光版并集(positionWS/normalWS/screenPos/fog)；frag `clip` 后 `#if defined(_UNLIT_ON) 输出染色+淡出+雾 #else BlinnPhong(+草的 _ShadowGIStrength 压暗环境光) #endif`；删掉 `*_Unlit.shader`(先确认 0 材质引用)、`*_Lit.shader` 去后缀重命名(GUID 随 .meta 保留→材质不断链)。②Mesh 草/树(TransparentCutout,已被 11 个 Building 材质引用,本就受光)**原地加** `_UNLIT_ON`(默认1=受光)：frag `clip` 后包 `#if defined(_UNLIT_ON) MixFog+return #else 原 BlinnPhong #endif`，ShadowCaster/DepthOnly 不动。③Sprite 草/树(本是无光,无材质引用)默认 `_LitEnable=0`(无光,保持原样),补 normalOS/positionWS/normalWS + include CommonLit,受光分支走 `ApplyCommonLit`。**注意 keyword 约定不一致**：`*_Common_1`(新建无材质)用 `_LIT_ON`(默认无光)；这批 WindSway(有材质须保受光)用 `_UNLIT_ON`(默认受光)——差异是迁移安全所迫,两者都复用同一 `ApplyCommonLit`(keyword 只是分支选择器)。`Assets/FrameWork` 是 **git 子模块**，删/改/重命名 shader 用文件系统操作(非外层 git),靠 Unity `Assets/Refresh` 由 .meta GUID 重建引用。

**三件套通用 shader `*_Common_1`(2026-07 新增, 描边+Lit+顶点位移旋转)**：`Shader_Particle_Common_1`(`Effect/`, "FrameWork/Particle/Common1")、`Shader_SpriteRenderer_Common_1`(`SpriteRenderer/`, "FrameWork/SpriteRenderer/Common1")、`Shader_Mesh_Common_1`(`URP/`, "FrameWork/URP/MeshCommon1")，分别用于粒子/SpriteRenderer/网格。三者实现：①描边(同上 `_OUTLINE_ON` 三件套 + `Common/Outline.hlsl`)②**Lit 开关做成单 shader 内的 `[Toggle(_LIT_ON)]_LitEnable` + `#pragma shader_feature_local _LIT_ON`**(不拆 Lit/Unlit 两文件——lit toggle 完全可在一个 shader 内 `#if defined(_LIT_ON) return ApplyCommonLit(...) #else 输出无光色+MixFog #endif` 分支实现)③**顶点位移+角度旋转(见下)**。正向 pass 常挂 URP 阴影/附加光 multi_compile(unlit 变体这些 keyword 只是不走分支)；粒子/精灵法线可能退化→length 太小退回视线方向再按 `IS_FRONT_VFACE` 双面翻转。渲染态：粒子=Transparent+Blend 可调+ZWrite 可调(单 forward pass)；精灵=Transparent+SrcAlpha OneMinusSrcAlpha+ZWrite Off(单 forward pass)；网格=TransparentCutout/AlphaTest+ZWrite On，另带 ShadowCaster+DepthOnly(按 alpha clip 投形状阴影)。CBUFFER inline 手写基础6字段(`_BaseMap_ST/_BaseMap_TexelSize/_BaseColor/_Cutoff/_OutlineColor/_OutlineSize`) + 位移旋转2字段——网格走 `HLSLINCLUDE` 共享给三 pass，粒子/精灵各自 pass 内声明。参数少故用 `[Header]` 分组(未挂 CustomEditor)。写完经 Unity MCP `Assets/Refresh` 导入，3 个均识别为 `UnityEngine.Shader`、0 console error。

**顶点位移+旋转件 `Common/Transform.hlsl`(2026-07 新增, 三件套 `*_Common_1` 全覆盖)**：跨效果通用顶点变换助手。`BuildEulerRotationMatrix(eulerDegrees)`→由欧拉角(度)构造物体空间 3x3 旋转矩阵(顺序 ZXY, R=Ry*Rx*Rz 与 Unity Transform 一致, 左乘列向量 `mul(R,v)`); `ApplyVertexTransform(positionOS, rotation, offset)`→先旋转后加偏移。接线：Properties 加 `[Header(Transform)] _VertexOffset("位置偏移(物体空间XYZ)",Vector) / _VertexRotation("角度旋转(欧拉角XYZ/度)",Vector)`(均默认0)；CBUFFER 加 `float4 _VertexOffset / float4 _VertexRotation`(Vector 属性映射 float4, 取 `.xyz`)；`#include "../Common/Transform.hlsl"`；每个 vert 里 `rotMat=BuildEulerRotationMatrix(_VertexRotation.xyz); positionOS=ApplyVertexTransform(IN.positionOS.xyz,rotMat,_VertexOffset.xyz); normalOS=mul(rotMat,IN.normalOS)` 再喂 `GetVertexPositionInputs`/`TransformObjectToWorldNormal`。**要点**：默认全0=恒等变换零成本外观不变；旋转绕物体原点(pivot)非几何中心；法线须同矩阵旋转否则 Lit 受光错乱；**网格三 pass(Forward/ShadowCaster/DepthOnly)必须同步变换**否则阴影/深度轮廓与可见网格错位(DepthOnly 只有 position 无 normal, 只转 position)。默认0时旋转矩阵仍每顶点算一次 sincos(可接受, 顶点着色器开销小)。**随时间自动旋转扩展(2026-07, 目前仅 `Shader_Mesh_Common_1` 接入)**：Transform.hlsl 加 `ApplyTimeRotationEuler(staticEuler, speed, direction, time)`=在静态欧拉角上叠加"每轴角速度(度/秒)×方向(±1)×_Time.y";shader 加 `[Header(Auto Rotate)] [Toggle(_ROTATE_TIME_ON)]_AutoRotateEnable / [Enum(Forward,0,Reverse,1)]_RotateDirection(枚举标签须 ASCII 否则 ShaderLab 报 $undefined,着色器里 1-2*x 换算 ±1) / _RotateSpeed(Vector 每轴度每秒,默认全 0=默认不转)` + CBUFFER `float4 _RotateSpeed / half _RotateDirection` + 三 Pass 各 `#pragma shader_feature_local _ROTATE_TIME_ON`;vert 里 `_ROTATE_TIME_ON` 开时把 `_VertexRotation.xyz` 换成叠加时间的欧拉角再喂 `BuildEulerRotationMatrix`(Forward/ShadowCaster/DepthOnly 同步→旋转的阴影/深度跟随)。`_Time` 是 Core.hlsl 全局非 UnityPerMaterial 不破 SRP Batcher。GUI `MeshCommonShaderGUI` 加 `自动旋转` Section(toggle `_AutoRotateEnable` + `_RotateDirection`/`_RotateSpeed`,关闭时置灰)。

**通用渲染设置件 `Common/SurfaceOptions.hlsl` + 面板助手 `Shader/Editor/SurfaceOptionsGUI.cs`(2026-07 新增)**：把"表面类型/渲染模式(混合)/Alpha 裁剪/渲染面"四项合成一个可复用模块，任意 shader 均可接入。
- **HLSL 件**：`SURFACE_OPTIONS_CBUFFER` 宏(仅 `half _Cutoff;`，块首展开保 SRP Batcher) + `ApplyAlphaClip(alpha, cutoff)`(参数化 cutoff **避免 include 早于 CBUFFER 的前向引用**，`#if _ALPHATEST_ON clip(alpha-cutoff)`)。表面类型/渲染模式/渲染面属固定功能态(Blend/ZWrite/Cull)不进 HLSL，只由材质属性+面板助手驱动，故 HLSL 件只封装需在 frag 运算的 Alpha 裁剪。
- **面板助手**(静态类，非基类，宿主 GUI 调用即可)：`SurfaceOptionsGUI.Draw(editor, props, drawn, foldout)` 画"渲染设置"折叠组(表面类型→渲染模式[不透明时置灰]→Alpha 裁剪开关+阈值[关时置灰]→渲染面，隐藏的 _SrcBlend/_DstBlend/_ZWrite 一并登记进 drawn 防落"其他"组；foldout 传宿主折叠器保风格统一)；`SurfaceOptionsGUI.Sync(editor, props, ref lastStateKey)` 把预设同步为实际渲染态——不透明=`One Zero`+写深度+(裁剪则 AlphaTest/TransparentCutout 队列 否则 Geometry/Opaque)、透明=按渲染模式取混合因子+不写深度+Transparent 队列；src/dst/zwrite 每帧幂等(仅值变才写)，队列/RenderType 仅"渲染状态签名(表面*10+裁剪)"切换时重设(首帧 NaN 不触发→保留手动队列/不clobber老材质)。
- **接线三件套**：①Properties 声明 `[Enum(Opaque,0,Transparent,1)]_Surface / [Enum(AlphaBlend,0,Additive,1,PremultipliedAlpha,2)]_BlendMode / [Toggle(_ALPHATEST_ON)]_AlphaClip / _Cutoff(Range0-1) / [Enum(On,0,Off,2)]_Cull / [HideInInspector]_SrcBlend=5/_DstBlend=10/_ZWrite`；②Pass 用 `Blend [_SrcBlend] [_DstBlend]` `ZWrite [_ZWrite]` `Cull [_Cull]` + `#pragma shader_feature_local _ALPHATEST_ON` + `#include "../Common/SurfaceOptions.hlsl"` + CBUFFER 内 `SURFACE_OPTIONS_CBUFFER` + frag 输出前 `ApplyAlphaClip(a, _Cutoff)`；③CustomEditor 的 OnGUI 里 `SurfaceOptionsGUI.Draw(...)` + `SurfaceOptionsGUI.Sync(..., ref lastKey)`(宿主持 `float lastKey=NaN` 字段)。首个接入者 `Shader_Mesh_ProgressBar`(MeshProgressBarShaderGUI 已从自带 Section 迁到调用助手)。ShaderLab Properties 仍无法 include→每 shader 自留一份属性声明(约定同上)。
- **`Sync` 的 `manageQueue` 形参(2026-07)**：宿主若自带队列控制(如 MeshCommon1 的"优先级"滑条)应传 `manageQueue:false`——Sync 仍每帧同步 `_SrcBlend/_DstBlend/_ZWrite` 与切换时的 `RenderType` 标签, 但**不重设 renderQueue**, 避免把滑条设的队列(如地毯3000)清掉; 默认 `true`(ProgressBar 用, 队列随表面类型归 Geometry/AlphaTest/Transparent)。
- **`Draw` 的"切不透明默认开 Alpha 裁剪"(2026-07, 通用行为)**：`SurfaceOptionsGUI.Draw` 在 `_Surface` 下拉 `BeginChangeCheck`, 一旦改到 Opaque(0) 就调 `SetAlphaClip(editor,alphaClip,true)`(同步 `_AlphaClip.floatValue=1` + 各材质 `EnableKeyword("_ALPHATEST_ON")`, 二者必须一起设否则面板值与变体脱节)——本项目不透明网格多为镂空 cutout, 需按轮廓裁剪; 只在切换瞬间置默认, 用户仍可手动再关。此为共享助手行为, ProgressBar 亦受益。
- **MeshCommon1 接入 SurfaceOptions(2026-07, 第二个接入者)**：`Shader_Mesh_Common_1` 原为固定 TransparentCutout+硬编码 `clip`+`ZWrite On`, 现接入表面渲染通用参数 → **表面类型可设不透明/透明**。三 Pass(Forward/ShadowCaster/DepthOnly)均加 `#pragma shader_feature_local _ALPHATEST_ON` + `ApplyAlphaClip` 替换硬 clip(SurfaceOptions.hlsl 提进 `HLSLINCLUDE` 供三 pass 共用; `_Cutoff` 仍手写在共享 CBUFFER 不重复用 `SURFACE_OPTIONS_CBUFFER` 宏), Forward 加 `Blend [_SrcBlend] [_DstBlend]`+`ZWrite [_ZWrite]`, Lit 分支 alpha 由 `1.0` 改 `col.a`。`_AlphaClip` 默认值 `0→1`(默认表面 Opaque, 故默认开裁剪)。GUI `MeshCommonShaderGUI` 从"base.OnGUI+滑条"重写为 Section 分组(表面/光照/描边/变换)+`SurfaceOptionsGUI.Draw`(渲染设置组)+`Sync(manageQueue:false)`+保留"优先级"滑条(队列仍 `2450+offset` 独占)。27 个引用材质(Building 石×22/墙/柱/宝座/地毯/BG)**本就序列化了** `_Surface=0/_SrcBlend=1/_DstBlend=0/_ZWrite=1/_Cull`(URP 标准表面属性集)但 `_AlphaClip=0`+空关键字表——老 shader 一直无条件硬 clip, 若只 honor `_AlphaClip=0` 会让带 alpha 镂空的建筑贴图**变整片方块(回归)**。故用 MCP `execute_code` 批量把 27 个不透明材质 `_AlphaClip=1`+`EnableKeyword("_ALPHATEST_ON")`, 恢复原镂空(对纯不透明贴图是 no-op, 安全)。

`DrawMeshInstanced` **不提供** `unity_MatrixPreviousM`(那是 MeshRenderer 运动矢量用的)。"火球过去在哪"这个信息不存在, 故速度必须由外部给。**默认落地方式=材质级常量(零 C# 改动)**: `_SparkVelocity` 虽声明为逐实例属性, 但**逐实例属性在没有 MPB 灌数组时会回退读材质上的值**, 故直接在材质面板「火星世界拖拽速度」填上该弹道的实际飞行速度即可生效, `AttackModeInstanceRenderer`/`BaseAttackMode` **一个字都不用改**(2026-07 用户明确要求: 不许往这两个文件加火球相关代码)。⚠️代价=**整桶一个速度**: 同一 `visual_name` 被正反方向弹道共用时, 只有一个方向对(与 `trail_data` 的 `color` 在方案1 下是桶级同理)。真要逐发不同速度才需渲染器按实例 `MPB.SetVectorArray` 灌(先例=轨迹的 `_TrailAlpha` `SetFloatArray`)。**已验证**: ①MPB 灌入路径——火球固定原点只改速度, 火星亮度重心 X 由 254.1(速度0)→232.9(+X 2)→197.3(+X 6) 单调左移、呈彗星尾; 单发预览可不灌(默认0)。渲染态=透明+加法叠加(SrcAlpha One)+ZWrite Off+Transparent 队列, 单 forward pass(不做 ShadowCaster/DepthOnly: 顶点位置是 shader 算的深度 pass 也对不上)。刻意**不复用 `ParticleCommon.hlsl`**(软粒子要采场景深度图, 而加法混合的小亮点与场景相交本就不穿帮), 粒子模拟逻辑 inline 在 .shader 内(仅 1 个 shader 用→按"被复用≠通用"判定不进 Common/Effect)。挂 `CustomEditor "MeshCommonShaderGUI"`, 该 GUI 的 `sections` 已加"中心火球(toggle `_CoreEnable`)/火星运动/火星外观"三组(组内属性不存在时整组自动跳过→ MeshCommon1 材质面板不受影响, 该数组=全部宿主 shader 的参数并集)。**⚠️Inspector 的分组显示顺序由 `MeshCommonShaderGUI.sections` 数组顺序决定, 不是 shader 的 Properties 块顺序**(挂了 CustomEditor 就轮不到 Properties 排版说话)——要调面板上下顺序改那个数组, Properties/Header 排列同步跟着改只是为了读代码时一致。当前约定=**火球在上、火星在下**。**配套网格工具** `FireBallMeshGeneratorWindow`(`Editor/Base/Window/`, 菜单 `Custom/工具弹窗/火球网格生成器`)：生成火球网格(火星数量/**包含中心火球**开关(核心 quad 烤在最后)/喷射角度(360=全向球状, 小于360=绕中心轴锥形, `cosθ` 均匀采样避免轴心过密)/逐火星速度·大小·生命倍率随机范围/包围盒半径/随机种子), 覆盖同路径资源时 `CopySerialized` 复用 GUID 不断引用。**已验证**: Unity 6000.3.11f1 导入 `ShaderHasError=False`/`MessageCount=0`/`isSupported=True`, 临时相机(LDR RT)渲染: 纯火星版 3451 亮像素、加核心后 9383 亮像素(核心=噪声撕出火舌的翻滚本体+橙红边缘色温, 火星=大而亮黄白新生 + 小而暗红将熄, 色温/尺寸衰减/边缘撕裂均生效)。预览资产在 `Assets/Out/FireSparkPreview/`(网格/软圆点贴图/材质, 用完可删)。**HDR 提示**: `_SparkColorStart`(4.0,2.2,0.6)/`_CoreColorHot`(5.0,3.0,1.0) 是给 HDR+Bloom 过曝辉光用的, LDR 下会截断成偏黄——用临时 RT 截图验证时别误判成"颜色错了"。
